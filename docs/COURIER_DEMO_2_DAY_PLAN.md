# TajGo Courier Demo — план на 2 дня

## Текущее состояние после аудита

- Android debug APK собирается.
- `flutter analyze` проходит без замечаний.
- Автотесты проходят.
- Firebase инициализируется через FlutterFire; используется анонимная авторизация.
- Firestore содержит репозитории пользователей, курьеров и заказов.
- Клиент может выбрать точки на OpenStreetMap и создать заказ.
- Курьер может выйти на линию, передавать GPS, принять/отклонить заказ и менять его статус.
- Экран выбора роли восстановлен и сохраняет клиентский и курьерский сценарии.

## День 1 — стабильный сценарий курьера

1. Прогнать на двух устройствах или в двух сессиях сценарий «клиент создаёт → курьер принимает».
2. Добавить карту активного заказа курьеру: получение, доставка, маршрут и кнопка внешней навигации.
3. Исправить обработку Firestore-индексов и ошибок сети понятными сообщениями.
4. Зафиксировать безопасные demo rules и необходимые индексы в репозитории.

## День 2 — версия для показа

1. Завершить цепочку статусов: `waiting → accepted → pickedUp → delivered`.
2. Показать заработок и историю завершённых заказов.
3. Проверить разрешения GPS, потерю сети, повторный запуск и конкурирующее принятие заказа.
4. Собрать и установить APK, пройти демонстрационный чек-лист, сделать release checkpoint.

## До публичного запуска обязательно

- зарегистрировать финальный package `tj.tajgo.app` как новый Android app в Firebase и заменить `google-services.json`;
- закрыть тестовые Firestore rules;
- перейти с публичного OSM tile-сервера на коммерчески допустимого поставщика или собственные тайлы;
- заменить анонимный вход на телефонную авторизацию и внедрить проверку ролей на сервере.
