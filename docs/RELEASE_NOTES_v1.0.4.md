# TajGo v1.0.4 — Map Accuracy + Controls Polish

## Исправленная критичная проблема

`pricing.haversineDistanceKm()` использовал `latlong2 Distance()` с включённым
по умолчанию `roundResult`. Пересчёт выполнялся сразу в километрах, поэтому
результат округлялся до целого километра: расстояния меньше примерно 500 м
становились `0.0 км`. Затем direct fallback и route summary принимали этот
ноль за реальную дистанцию и показывали «менее 10 м». Ответ road provider и
cache также не сравнивался с независимым расстоянием между A и B.

В v1.0.4 direct distance рассчитывается через `DistanceHaversine` с
`roundResult: false`, сохраняя дробные километры и метры.
`RouteSanityService` проверяет геометрию и расстояние provider относительно
baseline. Невозможный ответ незаметно заменяется direct fallback.

## Правила sanity-check

- маршрут отсутствует или содержит меньше двух точек;
- расстояние не положительное;
- provider меньше 60% прямого расстояния;
- provider больше прямого расстояния в пять раз;
- baseline больше 50 м, а provider сообщает меньше 10 м;
- разные A/B дают маршрут короче 20 м.

При таком результате пользователь видит «Маршрут предварительный», точное
прямое расстояние и приблизительный ETA, но не техническую ошибку.

## UX карты

- кнопки обзора маршрута и геолокации находятся справа над панелью;
- панель деталей на 360×800 занимает не больше 360 px;
- кнопка создания заказа закреплена снизу панели;
- route summary показывает расстояние, ETA, качество и «Показать весь»;
- адреса сокращаются до двух человеческих строк;
- Plus Code и повтор города/страны не являются основным адресом;
- fullscreen-структура карты из v1.0.3 сохранена.

## Debug-диагностика

Только в debug-сборке логируются подтверждённые A/B, источник точки,
baseline, provider distance, выбранное расстояние, качество, provider, число
точек, cache hit/miss и причина fallback.

## Полевой запуск

```powershell
flutter run --dart-define-from-file=tool/routing.local.json
```

Проверить две заведомо разные точки сначала с road provider, затем с
`ROUTING_ENABLED=false`.
