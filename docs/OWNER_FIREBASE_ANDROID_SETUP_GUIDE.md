# TajGo — инструкция владельца: регистрация Android-приложения в Firebase

Простая пошаговая инструкция. Понадобится: компьютер с проектом
(C:\Projects\tajgo), интернет, аккаунт Google, под которым создан проект
Firebase, телефон.

**Важно перед началом:** в коде проекта уже должен стоять новый package
`tj.tajgo.app` (это делает Codex). Если не уверены — спросите, потом
возвращайтесь сюда.

Вся процедура занимает 15–20 минут.

---

## Шаг 1. Откройте Firebase Console

1. В браузере откройте: **https://console.firebase.google.com**
2. Войдите под тем же Google-аккаунтом, на котором создан проект TajGo.
3. Кликните на карточку проекта TajGo.

## Шаг 2. Зайдите в настройки проекта

1. Слева вверху, рядом с надписью «Project Overview», есть **шестерёнка ⚙️**.
2. Нажмите её → выберите **«Project settings»** (Настройки проекта).
3. Останьтесь на первой вкладке **«General»** (Общие).

## Шаг 3. Добавьте Android-приложение

1. Прокрутите страницу вниз до раздела **«Your apps»** (Ваши приложения).
2. Нажмите кнопку **«Add app»** → выберите значок **Android** (робот).
3. Откроется форма регистрации — заполняем её в шагах 4–5.

## Шаг 4. Package name — самое важное поле

В поле **«Android package name»** введите ровно, без пробелов и заглавных
букв:

```
tj.tajgo.app
```

⚠️ Проверьте дважды. Одна опечатка — и приложение не свяжется с Firebase.
Исправить package потом нельзя — только удалять и добавлять заново.

## Шаг 5. App nickname

В поле **«App nickname»** введите:

```
TajGo Android
```

Это просто название для вас, на работу не влияет.

## Шаг 6. Что такое SHA-1 и SHA-256 (простыми словами)

У каждой сборки приложения есть «отпечаток пальца» — длинная строка из букв
и цифр. По нему Google проверяет: «это настоящее приложение TajGo, а не
подделка». Без отпечатка не заработает вход по SMS.

Отпечатков два (SHA-1 и SHA-256) — это просто короткая и длинная версия
одного и того же. Добавим оба.

## Шаг 7. Получите отпечатки (одна команда)

1. На компьютере нажмите **Пуск**, наберите **PowerShell**, откройте его.
2. Скопируйте и вставьте команду целиком, нажмите Enter:

```powershell
keytool -list -v -alias androiddebugkey -keystore "$env:USERPROFILE\.android\debug.keystore" -storepass android -keypass android
```

3. В ответе найдите строки:
   - `SHA1: A1:B2:C3:...` — это SHA-1;
   - `SHA256: D4:E5:F6:...` — это SHA-256.
4. Выделите каждую строку значений (всё после «SHA1:») и скопируйте.

**Если пишет «keytool не является командой»** — выполните вместо этого:

```powershell
cd C:\Projects\tajgo\android
.\gradlew signingReport
```

и найдите в конце вывода блок `Variant: debug` — там будут те же SHA1 и
SHA-256.

## Шаг 8. Вставьте отпечатки в Firebase

1. Вернитесь в форму из шага 3. Под полем package есть
   **«Debug signing certificate SHA-1»** — вставьте туда SHA-1.
2. Нажмите **«Register app»** (Зарегистрировать).
3. SHA-256 добавляется после регистрации: Project settings → «Your apps» →
   карточка TajGo Android → кнопка **«Add fingerprint»** → вставьте SHA-256
   → Save.

Двоеточия в строке оставляйте как есть — так и надо.

## Шаг 9. Скачайте google-services.json

После регистрации Firebase предложит скачать файл
**google-services.json** — нажмите **«Download google-services.json»**.

Если окно уже закрыли: Project settings → «Your apps» → TajGo Android →
кнопка **«google-services.json»** со стрелкой вниз.

⚠️ Убедитесь, что в карточке, откуда качаете, написано именно
`tj.tajgo.app` — если в проекте было старое приложение, файлов может быть
два, нужен новый.

## Шаг 10. Положите файл в проект

1. Найдите скачанный файл (обычно папка «Загрузки»).
2. Скопируйте его сюда, заменив старый файл:

```
C:\Projects\tajgo\android\app\google-services.json
```

Windows спросит «Заменить файл?» — да, заменить.

## Шаг 11. Пересоберите приложение

Подключите телефон кабелем и выполните в PowerShell три команды по очереди
(каждую — Enter, дождаться окончания):

```powershell
cd C:\Projects\tajgo
flutter clean
flutter pub get
flutter run -d 26993bd3
```

Последняя команда соберёт и запустит TajGo на телефоне. Первая сборка после
clean — долгая, 3–5 минут, это нормально.

## Шаг 12. Если приложение не ставится поверх старого

Package сменился — для телефона это «другое приложение», и старое может
мешать. Решение:

1. На телефоне: удалите старое приложение TajGo (зажать иконку → Удалить).
2. Снова выполните `flutter run -d 26993bd3`.

## Шаг 13. Что пока НЕ трогать

- **Production keystore** (боевой ключ подписи) — понадобится только перед
  публикацией в магазине.
- **Play Store / публикацию** — отдельный этап.
- **Firestore deploy** (кнопки «Publish rules» в консоли) — правила деплоим
  отдельно и осознанно, не сейчас.

Если консоль предлагает что-то ещё «включить/обновить» — пропускайте.

## Шаг 14. Частые ошибки

| Что видите | Причина и решение |
|---|---|
| Ошибка со словами `package name ... does not match` | В Firebase введён не тот package. Сверьте: должно быть ровно `tj.tajgo.app`. Если опечатка — удалите приложение в консоли и добавьте заново (шаги 3–10) |
| SMS-код не приходит, ошибка при входе по телефону | Чаще всего не добавлен SHA (шаги 7–8) — добавьте оба отпечатка и пересоберите (шаг 11) |
| Ошибка `operation-not-allowed` при входе | Phone Auth не включён: в консоли слева **Authentication** → вкладка **Sign-in method** → **Phone** → включить → Save |
| Приложение падает сразу при запуске после замены файла | Скачан не тот google-services.json (от старого приложения). Скачайте из карточки `tj.tajgo.app` (шаг 9) и повторите шаги 10–11 |
| `flutter run` пишет «No devices found» | Телефон не виден: проверьте кабель; на телефоне включите «Отладку по USB» (Настройки → Для разработчиков) и подтвердите запрос «Доверять этому компьютеру». Проверить: `flutter devices` |
| Windows ругается на `cd` или команды не находятся | Убедитесь, что вы в PowerShell (синее окно), а не в другом терминале, и что первая команда была `cd C:\Projects\tajgo` |

**Как понять, что всё получилось:** приложение запустилось на телефоне,
сплэш прошёл без ошибки «Не получилось подключиться», а вход по номеру
телефона присылает SMS (или принимает тестовый номер). Если так — Firebase
и Android подружились, можно звать тестеров. 💚
