# Импорт бренд-ассетов TajGo

Приложение использует только выбранные владельцем изображения из каталога
`assets/brand/`. Автоматически создавать или подменять эти изображения не нужно.

## Обязательные имена

| Путь | Точный размер | Содержимое |
| --- | ---: | --- |
| `assets/brand/app_icon.png` | 1024×1024 px | Выбранная иконка приложения, sRGB, непрозрачный PNG. |
| `assets/brand/splash_light.png` | 1080×1920 px или больше | Выбранная светлая заставка. |
| `assets/brand/splash_dark.png` | 1080×1920 px или больше | Выбранная тёмная заставка. |
| `assets/brand/tajgo_logo.png` | 1024×1024 px | Выбранный логотип TajGo на прозрачном фоне. |

Все файлы должны использовать цветовой профиль sRGB. SplashScreen показывает
выбранный фон на весь экран, логотип сверху и только служебный индикатор загрузки
снизу. Дополнительные горы, иллюстрации, название и слоган код больше не рисует.

## Безопасная замена

1. Сохраните новые PNG под точными именами выше.
2. Не меняйте пути в коде и не удаляйте каталог `assets/brand/`.
3. Выполните:

```powershell
flutter pub get
dart run flutter_launcher_icons
flutter analyze
flutter test
flutter build apk --debug
```

4. Удалите старую debug-установку с телефона и установите новый APK, иначе launcher может показывать закэшированную иконку.

Если изображение отсутствует, приложение не создаёт вместо него новую графику:
будет показан только нейтральный фон. Поэтому перед сборкой проверьте наличие всех
четырёх файлов.
