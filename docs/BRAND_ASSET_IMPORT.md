# Импорт бренд-ассетов TajGo

Приложение ожидает брендовые PNG в каталоге `assets/brand/`. В v0.8.3 все перечисленные файлы уже добавлены. Эта инструкция нужна для безопасной замены исходников в будущем.

## Обязательные имена

| Путь | Точный размер | Содержимое |
| --- | ---: | --- |
| `assets/brand/tajgo_logo.png` | 1024×1024 px | Полноцветный знак TajGo без скругления углов. |
| `assets/brand/app_icon.png` | 1024×1024 px | Мастер launcher-иконки, sRGB, непрозрачный PNG. |
| `assets/brand/app_icon_foreground.png` | 1024×1024 px | Знак на прозрачном фоне для adaptive icon. |
| `assets/brand/app_icon_background.png` | 1024×1024 px | Непрозрачный фон adaptive icon. |
| `assets/brand/app_icon_monochrome.png` | 1024×1024 px | Одноцветный белый знак на прозрачном фоне. |
| `assets/brand/splash_light.png` | 1080×1920 px | Светлая вертикальная панорама без текста и логотипа. |
| `assets/brand/splash_dark.png` | 1080×1920 px | Тёмная вертикальная панорама без текста и логотипа. |

Все файлы должны использовать цветовой профиль sRGB. Splash-фоны оставляют свободной верхнюю часть: логотип, название, слоган и индикатор рисует Flutter, поэтому текст не должен быть встроен в изображение.

## Безопасная замена

1. Сохраните новые PNG под точными именами выше.
2. Не меняйте пути в коде и не удаляйте каталог `assets/brand/`.
3. Выполните:

```powershell
flutter pub get
dart run flutter_launcher_icons
flutter analyze
flutter test
flutter build apk --debug
```

4. Удалите старую debug-установку с телефона и установите новый APK, иначе launcher может показывать закэшированную иконку.

## Fallback

Если `tajgo_logo.png`, `splash_light.png` или `splash_dark.png` не загрузятся, SplashScreen не падает: вместо изображения используется кодовый знак `T+`, градиент и нарисованный силуэт Худжанда. Launcher-иконки, напротив, генерируются заранее, поэтому удалять их исходники перед запуском `flutter_launcher_icons` нельзя.
