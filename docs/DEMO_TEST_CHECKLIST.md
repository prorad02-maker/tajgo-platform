# TajGo v0.7.0 — чеклист демо

## Подготовка

- Установить один и тот же debug APK на два Android-телефона.
- Телефон 1: клиент. Телефон 2: курьер. Разрешить GPS и включить интернет.
- Для реального админ-входа установить `users/{uid}.role = "admin"` вручную.
- До deploy admin Rules админские действия ожидаемо вернут `permission-denied`.

## Полный сценарий

1. Клиент открывает «Доставка», выбирает A и B и создаёт заказ.
2. Курьер открывает «Стать курьером» и выходит на линию.
3. Курьер принимает заказ и видит себя, A/B, расстояние и ETA.
4. Клиент открывает заказ и видит live-маркер из `courier_public`.
5. В радиусе 2 км курьер нажимает «Я на месте», затем «Забрал заказ».
6. Цель навигации меняется с A на B.
7. Курьер нажимает «Передал заказ» и вводит четырёхзначный код клиента.
8. Статус становится `completed`, статистика курьера обновляется.
9. Администратор открывает «🛠 Управление TajGo» и проверяет заказ, курьера и карту.

## Быстрая подготовка через Demo Tools

В debug-сборке на главной открыть «Тест · Demo Tools»:

- создать обычный или ближний тестовый заказ;
- сбросить застрявший `activeOrderId`;
- вернуть выбранный заказ в `waiting`;
- отменить waiting-заказы с меткой `[TEST]`;
- посмотреть UID, роль, телефон и online-состояние.

Все тестовые действия требуют подтверждения. В release экран отсутствует.

## Если демо застряло

1. Проверить GPS и разрешения обоих телефонов.
2. В Demo Tools сбросить активный тестовый заказ.
3. Если secure Rules блокируют admin action — выполнить операцию вручную в Firebase Console либо сначала согласованно задеплоить Rules.
4. Не удалять production-документы из Demo Tools.
