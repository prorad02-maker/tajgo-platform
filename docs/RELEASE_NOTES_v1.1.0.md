# TajGo v1.1.0 — Account Foundation, Auth & Modes

## Что готово

- Splash передаёт управление единому App Startup Router.
- Новый пользователь выбирает намерение: заказать доставку или зарабатывать с
  TajGo. Пользовательского выбора admin/«роли» больше нет.
- Phone Auth поддерживает номер `+992`, SMS-код, автоподтверждение, повторную
  отправку через 60 секунд и русские тексты ошибок.
- Debug-сборка сохраняет явно помеченный «Тестовый вход»; в release он скрыт.
- Anonymous-пользователь сначала связывается с телефоном через
  `linkWithCredential`, сохраняя UID. Конфликт существующего телефона требует
  явного подтверждения и ничего не удаляет автоматически.
- `users/{uid}` получил `roles`, `lastMode`, `courierStatus`, `phoneVerified`,
  `profileComplete` и совместимость с legacy-полями.
- Клиентский режим доступен всегда. Курьерский — только после `approved`, без
  повторного SMS и без создания второго аккаунта.
- Добавлены общий профиль, настройки, выход без удаления Firestore-данных и
  снятие курьера с линии перед выходом/сменой режима.
- Ленивая миграция legacy-профиля идемпотентна и не трогает активный заказ,
  рейтинг, заработок и статистику.
- Release-заказ требует `phoneVerified=true`. Debug-anonymous заказ получает
  поле `isTestOrder=true` и видимый бейдж `ТЕСТ`.

## Совместимость

- Карта v1.0.4, routing, Firebase project, `firebase_options.dart` и
  `applicationId=tj.tajgo.app` не менялись.
- `couriers/{uid}` и `courier_public/{uid}` сохраняют прежнюю структуру.
- Локальные `firestore.rules` подготовлены для новых account-полей и тестового
  заказа, но **не развёрнуты**. Firebase deploy в этой задаче не выполнялся.
