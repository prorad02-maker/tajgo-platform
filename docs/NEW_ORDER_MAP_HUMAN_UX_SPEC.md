# TajGo — NewOrderMapScreen: человеческий UX выбора точек

Корректирующий UX-спек по скриншотам текущего экрана. Не редизайн приложения
— наведение порядка на одном экране. Без кода. RouteProvider, Firebase,
Phone Auth и бизнес-логику заказа НЕ трогаем — только UX экрана
NewOrderMapScreen и связанных map-виджетов.

Диагноз по скриншотам (кратко): экран выглядит технически — GPS-кнопка
подрезана панелью, две сущности «моё местоположение», Plus Code как адрес,
чёрный баннер поверх интерфейса, жёлтая «тревога» про приблизительный адрес,
пин сливается с синей точкой, панель съедает карту.

---

## A. Главный принцип

Экран отвечает пользователю ровно на три вопроса:

1. **Какую точку я сейчас выбираю?** (забор или доставка)
2. **Где находится выбранная точка?** (пин + короткий адрес)
3. **Что нажать, чтобы подтвердить?** (одна большая кнопка)

Всё, что не помогает ответить на эти три вопроса, — с экрана убрать.
Карта — главный элемент: минимум 50% экрана в обычном состоянии.

## B. Новая иерархия экрана

**Верх карты:**
- слева — круглая кнопка «назад»;
- справа/по центру — компактный чип курьеров, ТОЛЬКО если данные реальные
  (≥1 курьер) — иначе не показывать вовсе (см. §K).

**На карте:**
- центральный пин активной точки (цвет этапа: тёмно-зелёный A, лаймовый B);
- синяя точка пользователя (если гео известно);
- ОДНА кнопка геолокации справа, над панелью, всегда полностью видна;
- кнопка «Показать маршрут» — только на этапе деталей, когда A и B выбраны.

**Нижняя панель (компактная):**
- поля «Откуда» и «Куда» — по строке каждое, нажимаемые целиком;
- короткий адрес найденной точки (формат §E);
- одна контекстная подсказка (§G);
- одна главная кнопка (§F).

Больше ничего. Поиск, пин и геокнопка не конкурируют: поиск живёт в полях
панели, пин — на карте, геокнопка — просто «где я».

## C. Кнопка геолокации — одна, понятная

Убрать дублирование: текстовую кнопку «Использовать моё местоположение»
из панели УДАЛИТЬ. Механизм геолокации один — круглая кнопка на карте.
(«Откуда» по умолчанию и так предзаполняется текущим местоположением при
доступном гео — это не кнопка, а автоматика.)

Состояния кнопки:

| # | Состояние | Вид и поведение |
|---|---|---|
| 1 | Обычное | иконка прицела; при ПЕРВОМ открытии экрана — подпись-лейбл рядом «Показать, где я» (исчезает после первого тапа навсегда) |
| 2 | Определение | внутри кнопки маленький круговой индикатор; вторая кнопка НЕ появляется; повторный тап игнорируется |
| 3 | Найдено | карта плавно (≤1 сек) центрируется; синяя точка видна; короткая плашка у кнопки «Вы здесь» (2 сек) |
| 4 | GPS выключен | snackbar НАД панелью (не поверх кнопки): «Включите геолокацию, чтобы показать ваше положение.» + действие «Включить» |
| 5 | Permission denied | «Разрешите TajGo доступ к геолокации.» + действие «Разрешить» (повторный запрос) |
| 6 | Denied forever | «Геолокация запрещена в настройках.» + действие «Открыть настройки» |

Позиция: правый край, отступ 16, нижний край кнопки = верх панели + 12 px.
Кнопка привязана к панели, а не к экрану — панель выросла → кнопка уехала
вверх, но видна ВСЕГДА целиком.

## D. Пин и синяя точка — разные объекты

- **Синяя точка** = «где я» (факт). Маленькая, с белой обводкой, при слабом
  GPS — с кругом неопределённости.
- **Центральный пин** = «что я выбираю» (намерение). Крупный, цвета этапа,
  остриём в точку.

Они обязаны различаться размером, формой и цветом. Когда пин стоит ровно
над пользователем:
- пин визуально приподнят (остриё чуть выше точки, лёгкая тень) — синяя
  точка читается ПОД ним;
- в панели подпись: «Точка забора совпадает с вашим местоположением» —
  это подтверждение, а не проблема.

## E. Поля «Откуда» и «Куда»

Формат — две строки, компактно:

```
Откуда
Ваше местоположение          ← главный текст, w700
Худжанд                      ← вторая строка, мелко, muted
```

или после выбора конкретного места:

```
Откуда
ул. Исмоили Сомони, 54
Худжанд
```

Правила:
- **Plus Code («7JP6+86G…») НИКОГДА не главный текст.** Нет нормального
  адреса → главный текст «Точка на карте», вторая строка мелко — Plus Code
  или координаты (технический fallback, muted, 11px).
- Длинный адрес не обрезается многоточием в одну строку — короткое имя
  главной строкой, остальное во вторую.
- **Всё поле нажимаемо** (не только иконка): тап по «Откуда»/«Куда»
  открывает поиск/переводит на выбор этой точки.
- Заполненное поле показывает галочку-индикатор ✓ слева (точка выбрана).

## F. Главная кнопка

Одна большая лаймовая кнопка, тексты по этапам:

| Этап | Текст |
|---|---|
| Выбор A | «Подтвердить точку забора» |
| Выбор B | «Подтвердить точку доставки» |
| A и B выбраны (детали) | «Продолжить» → далее существующий «Найти курьера · N TJS» |

Требования:
- всегда в одном месте — низ панели, полная ширина, SafeArea учтён;
- НЕ исчезает никогда (нет условий «появляется иногда»); если подтверждать
  нельзя — disabled с видимой причиной строкой выше («Определяем адрес…»);
- snackbar/подсказки НЕ перекрывают кнопку (snackbar с floating behavior
  и отступом над панелью);
- disabled-вид: серо-мятный фон, текст muted — явно «не сейчас», а не
  «сломалось».

## G. Подсказки и предупреждения

**Чёрный постоянный баннер («Вы здесь. Точки Откуда и Куда выберите…») —
УДАЛИТЬ.** Вместо него — одна строка контекстной подсказки ВНУТРИ панели,
над кнопкой, muted, 12–13px:

- этап A: «Передвиньте карту или нажмите кнопку геолокации.»
- этап B: «Найдите адрес или передвиньте карту.»

Про приблизительный адрес: жёлтую тревогу «Адрес найден приблизительно» —
убрать из постоянного показа. Новый вариант:

- по умолчанию НЕ показывать ничего;
- только при действительно низком confidence геокодера — нейтральная
  подсказка: «Проверьте точку на карте» (muted, без жёлтого фона и ⚠).

Правило: на экране одновременно максимум ОДНА подсказка. Подсказка — это
шёпот, а не сирена.

## H. Размер нижней панели

- **Collapsed (обычный выбор): ~30–35% высоты экрана**, карта ≥ 50%.
- **Expanded — только при поиске адреса**: панель растёт, показывает
  результаты; закрытие поиска возвращает collapsed.
- Клавиатура: панель поднимается над клавиатурой, кнопка подтверждения
  остаётся видимой ИЛИ скрывается вместе с режимом поиска (тогда
  подтверждение результата — тап по результату). Ничего не «ломается»
  и не перекрывается.
- Панель НЕ перекрывает геокнопку никогда (§C — привязка кнопки к панели).

## I. Состояния экрана

| # | Состояние | Что видно | Главная кнопка | Основной текст | Что скрыто |
|---|---|---|---|---|---|
| 1 | Только открылся | карта на Худжанде, пин по центру, панель collapsed | «Подтвердить точку забора» (enabled после геокода) | «Где забрать?» + подсказка A | маршрут, чип курьеров (до данных) |
| 2 | GPS не запрашивался | то же, без синей точки | та же | та же | всё гео-специфичное; никаких упрёков |
| 3 | Нажал GPS | индикатор в геокнопке | та же (жива) | — | вторая кнопка/дубль |
| 4 | Гео найдено | камера у пользователя, синяя точка, «Вы здесь» (2 сек) | та же | «Откуда: Ваше местоположение» | плашка «Вы здесь» через 2 сек |
| 5 | Выбирается A | пин A по центру, адрес обновляется при остановке карты | «Подтвердить точку забора» | адрес по формату §E | поле B — видно, но неактивно-спокойно |
| 6 | A выбрана → B | пин стал лаймовым, маркер A остался на карте, поле A с ✓ | «Подтвердить точку доставки» | «Куда доставить?» + подсказка B | подсказка A |
| 7 | A и B выбраны | оба маркера, линия маршрута, панель деталей | «Продолжить» / «Найти курьера · N TJS» | цена, км, мин | пин-прицел по центру |
| 8 | Адрес не найден | пин на месте | enabled («Точка на карте» — валидный выбор) | «Точка на карте» + мелко координаты | ошибки/красный |
| 9 | Адрес приблизительный | пин на месте | enabled | адрес + «Проверьте точку на карте» (muted) | жёлтые баннеры |

| 11 | Тайлы грузятся | старые тайлы остаются, чип «Загружаем карту…» | enabled | — | размытие всей карты |
| 12 | Ошибка GPS | карта живёт | enabled | snackbar по §C-4/5/6 с действием | вторые кнопки, тревога |

Сквозное правило: главная кнопка видна во ВСЕХ 12 состояниях.

## J. Загрузка карты

- Старые тайлы остаются на экране, пока грузятся новые (никакого белого/
  размытого слоя на весь экран).
- Маленький чип «Загружаем карту…» у верхнего края — и всё.
- Интерфейс не блокируется: панель, поля, геокнопка живы.
- **Но:** «Подтвердить точку» и геокодирование НЕ срабатывают, пока камера
  в полёте анимации (центр карты в этот момент — случайная промежуточная
  точка). Подтверждение доступно после остановки камеры — это и сейчас
  логика MapEventMoveEnd, закрепить её и на программные анимации.

## K. Ограничение элементов

Плавающих элементов на карте одновременно максимум **три**:

1. кнопка «назад»;
2. кнопка геолокации;
3. центральный пин.

Чип «N курьеров рядом»: показывать только если N ≥ 1 (реальные данные) и
только на этапах выбора точек; «Пока нет курьеров рядом» на этом экране НЕ
показывать — это выбор адреса, а не витрина. Всё остальное — внутри панели.

## L. Эталонный конечный экран (wireframe)

```text
[←]                        [💚 3 курьера рядом]

                  К А Р Т А

                     📍  ← пин этапа (A: тёмно-зелёный)
                  ◉      ← синяя точка (если известна)

                                        [◎]  ← геокнопка, над панелью
──────────────────────────────────────────────
 Откуда   ✓ Ваше местоположение
           Худжанд
 Куда      Введите адрес
──────────────────────────────────────────────
 Передвиньте карту, чтобы уточнить точку

 [        Подтвердить точку забора        ]
──────────────────────────────────────────────
```

Карта ≥ 50%. Панель ~30–35%. Ничто ничего не перекрывает.

## M. Acceptance criteria для Codex

1. GPS-кнопка полностью видна на всех этапах.
2. GPS-кнопка не закрывается панелью (привязана к её верхнему краю).
3. На экране нет двух способов «использовать моё местоположение».
4. Синяя точка и пин визуально различимы (размер, форма, цвет).
5. Plus Code нигде не является главным адресом.
6. Главная кнопка всегда в одном месте и видна во всех состояниях §I.
7. Чёрный постоянный баннер удалён.
8. Карта занимает ≥ 50% экрана в collapsed-состоянии.
9. Нижняя панель не перекрывает ни одну кнопку.
10. «Приблизительный адрес» — muted-подсказка, не жёлтая тревога, и только
    при низком confidence.
11. После подтверждения A интерфейс явно переключился на B (цвет пина,
    заголовок, ✓ у поля A).
12. Человек, впервые открывший экран, понимает первое действие за 5 секунд.
13. Всё помещается на 360×800 без overflow (проверить на этом размере).
14. Проверено при системном масштабе текста 1.3.
15. Проверено в светлой и тёмной системной теме (сплэш-палитра не ломает
    экран; сам экран остаётся светлым — но без белых артефактов).
