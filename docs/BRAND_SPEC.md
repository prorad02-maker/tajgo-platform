# TajGo — Brand Layer v0.3.x: сплэш (light/dark), иконка, загрузка

Дизайн-спецификация + Flutter-ready инструкции для Codex. Функциональность,
архитектуру, Firebase, main.dart, навигацию и Courier MVP НЕ менять.

**Текущее состояние (не переделывать):** светлый сплэш уже реализован в
`lib/features/splash/splash_screen.dart` и принят: белый верх, `TajGoLogo`,
двухцветный «TajGo», слоган с зелёным «Для своих.», пейзаж Худжанда снизу
(`_KhujandLandscapePainter` — горы, солнце, город, холмы, дорога), скруглённая
полоса. Текст — Flutter-виджетами (как и требуется). Эта спека добавляет:
тёмный вариант, анимированную полосу и иконку.

## A. Концепция обновления бренда

Один сплэш — две палитры. Макет, композиция и код общие; светлая и тёмная
темы отличаются ТОЛЬКО набором цветов (палитра передаётся в painter и стили
текста). Днём — «утро в долине»: белый воздух, зелёный город. Ночью —
«вечерняя смена»: глубокий зелёно-чёрный, город силуэтом, тёплая луна,
лаймовые огни окон. Иконка — единый знак TG+ на пейзаже, с упрощёнными
слоями для adaptive/monochrome. Ничего нового в продуктовом flow.

## D. App Icon — 1024×1024, скруглённый квадрат

Утверждённая композиция (референс пользователя от 12.07.2026):

- Фон: вертикальный мягкий градиент неба `#F2F8DF → #CBE5AF`, солнце
  `#F7D765`, белое облако.
- Средний план: зелёный силуэт города/гор (`#6BA562` дальний, `#55964F`
  ближний) — крупные купола и минареты, БЕЗ мелочи.
- Центр: «TG» белые w900 (~34% высоты знака) + «+» лаймовый
  `#84CC16→#5FAE2E` с белым контуром. Мягкая тень под текстом.
- Низ: зелёный склон `#3F9250 → #1E6132`, белая S-дорога, слева курьер на
  скутере (белый контур, зелёный короб), справа пеший с коробкой, 1–2 листа.
- Тест читаемости: на 48×48 обязаны выживать ТОЛЬКО «TG+» и общая
  зелёная гамма; детали могут растворяться.

Комплект файлов (все PNG):

## E. Loading bar — поведение

Заменить текущий indeterminate `LinearProgressIndicator` на маленький виджет
**`TajGoProgressBar`** (локально в splash_screen.dart или shared):

- Размеры: ширина 132, высота 5, радиус 999.
- Фон: из палитры (§C). Заполнение: градиент `#16A34A → #84CC16`
  (Container + `FractionallySizedBox` по `AnimationController`;
  стандартный LinearProgressIndicator градиент не умеет).
- Анимация: `AnimationController` 1.5 сек, `Curves.easeOutCubic`,
  0 → 0.9 при старте экрана. Дальше:
  - bootstrap завершился раньше — доводим 0.9 → 1.0 за 200 мс и только
    потом навигация (заменить текущий `Future.delayed(700ms)` на «дождаться
    заполнения», минимум те же ~700 мс сохранить);
  - bootstrap ещё идёт — полоса стоит на 0.9 с лёгкой пульсацией opacity
    (0.7↔1.0, 1 сек, repeat) — «живая», но не врёт про прогресс;
  - ошибка — полоса останавливается, окрашивается в `TajGoColors.error`,
    статус-текст показывает ошибку (как сейчас).
- Под полосой — существующий динамический `_status` (12px). Отдельный текст
  «Загружаем TajGo...» не добавлять.

Почему не честный 0→100%: длительность bootstrap непредсказуема (сеть),
фиксированные 1.2–1.8 сек будут врать. Схема «быстро до 90% → ждём» —
стандарт (браузеры, nprogress).
