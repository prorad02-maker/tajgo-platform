# TajGo Map Architecture v1

## Цель

Карта в TajGo должна быть не просто экраном, а отдельным модулем, который можно использовать в разных сценариях:

- клиент выбирает адрес;
- курьер видит маршрут;
- админ видит курьеров онлайн;
- партнёр видит зоны доставки;
- система распределения заказов считает расстояние и время.

## Принцип

Карту не храним в базе данных.

В базе храним только бизнес-данные поверх карты:

- координаты курьера;
- координаты заказа;
- адреса;
- зоны доставки;
- статус маршрута;
- историю перемещений.

## Структура

```text
lib/features/map/
  models/
    tajgo_location.dart
  services/
    location_service.dart
  screens/
    tajgo_map_screen.dart
  widgets/
    tajgo_map_marker.dart
  map_module.dart
```

## Почему так

- UI карты не смешивается с разрешениями GPS.
- Геолокация вынесена в сервис.
- Координаты описаны отдельной моделью.
- Маркеры можно переиспользовать в клиенте, курьере и админке.
- Позже можно добавить маршрутизацию OSRM/GraphHopper без переписывания экрана.
