# TajGo — Customer MVP v0.4.0: продуктовая и UX-спецификация

Для внедрения ПОСЛЕ v0.3.1 Brand Polish. Дополняет `SPEC_CODEX_V3.md` +
`COURIER_UI_SPEC.md` (статусы completed/disputed, код подтверждения,
arrivedAtPickupAt — они общие для двух сторон). Дизайн — токены TG+
(`SPEC_CODEX_V1.md` §5): бело-зелёный, лаймовая главная кнопка, белые
карточки, мягкие тени, минимум текста.

**Уже реализовано и НЕ переделывается:** главная-платформа (шапка, модуль
«Доставка» с категориями, «скоро», «Стать курьером»), создание заказа через
карту (`NewOrderMapScreen`: пин-выбор Откуда/Куда, геокодирование, линия,
расчёт км/мин/цены, редактируемая цена), карточка активного заказа на
главной, отмена waiting-заказа, транзакции статусов.

---

## A. Концепция Customer MVP

Клиент оформляет доставку за 4 действия (тип → откуда → куда → подтвердить)
и дальше НЕ думает: приложение само показывает, что происходит, что делать
дальше и где курьер. Один активный заказ — один понятный экран отслеживания.
Прошлые заказы под рукой. Все цены — честные и объяснённые заранее.

## B. User flow

```text
Главная ──(категория или «Заказать доставку»)──► NewOrderMapScreen
   ▲                                                │ Откуда → Куда → детали
   │                                                │ (+комментарий, цена)
   │                                                ▼ «Найти курьера · N TJS»
   └──◄── OrderTrackingScreen ◄─────────────────────┘
          Ищем курьера… (отмена доступна)
          → «Курьер найден!» (имя, рейтинг)
          → курьер на карте, статусы по шагам
          → pickedUp: КОД крупно
          → delivered: «Получил» / «Не получил»
          → completed: «Спасибо!» → Главная (история пополнилась)
```

## C. Список экранов

### C1. CustomerHomeScreen (доработать)

Сверху вниз:
1. Шапка TG+ (есть). Приветствие: «Добро пожаловать 💚» / по времени суток

4. **НОВОЕ — «Мои заказы»**: последние 5 завершённых/отменённых заказов
   (completed/cancelled/disputed) компактными строками: тип-эмодзи,
   «A → B» (1 строка, ellipsis), цена, бейдж статуса (✅ mint / ✖ серый /
   ⚠ warning), дата «сегодня/вчера/12.07». Пусто → секцию скрыть.
5. Ряд «скоро» и «Стать курьером» (есть).

### C3. OrderTrackingScreen — НОВЫЙ экран отслеживания

Файл: `lib/features/customer/order_tracking_screen.dart`. Открывается после
создания заказа и тапом по карточке активного заказа. Слушает
`orderRepository`-стрим заказа (переиспользовать activeOrderStream или
добавить `orderStream(orderId)` в OrderRepository — как в CourierRepository).

- **Карта** (верхние ~55%): пины A (darkGreen) и B (lime), линия маршрута;
  с момента accepted — **живой маркер курьера**: StreamBuilder на
  `courierRepository.courierStream(order.courierId)`, зелёная точка с белой
  обводкой по `couriers.location` (поле уже пишется курьером). До accepted
  маркера нет.
- **Нижняя панель** (белая, скругление 20): статусный блок по §D + карточка
  заказа (тип, A → B, цена, комментарий) + `TajGoOrderProgress` (шаги:
  Поиск → Принят → Забрал → Доставлено).

## D. Состояния экранов (OrderTrackingScreen / карточка на главной)

| Статус | Заголовок | Что ещё видит клиент | Действия |
|---|---|---|---|
| waiting | «🔎 Ищем курьера…» | пульсирующая точка/индикатор; «Обычно это занимает пару минут» | «Отменить заказ» (вторичная; транзакция только из waiting — есть) |
| accepted | «🚴 Курьер найден!» | имя и рейтинг курьера (из couriers/{courierId}: name, rating ⭐), маркер на карте; баннер «Курьер найден!» показывается 3 сек при смене waiting→accepted | — |
| accepted + arrivedAtPickupAt | «📍 Курьер на месте забора» | то же | — |
| pickedUp | «📦 Заказ у курьера» | **КОД крупно** (см. §F-код), курьер едет к вам, маркер движется | — |
| delivered | «Курьер передал заказ?» | пояснение «Подтвердите получение» | «✅ Получил» (лайм) / «Не получил» (текстовая, error-цвет, с диалогом «Вы уверены?») |
| completed | «✅ Доставлено. Спасибо!» | итог: цена, время; авто-возврат на главную через 3 сек или кнопка «На главную» | — |
| disputed | «⚠️ Мы разбираемся» | «Заказ помечен как неполученный. Мы свяжемся с вами.» | — |
| cancelled | карточка/экран закрывается | — | — |

## F. Firestore — дельта для клиентской части

| Поле | Где | Описание |
|---|---|---|
| `comment` | orders | String?, ≤200 символов, из формы details; показывается курьеру в карточке заказа и на экране его активного заказа |

Остальное уже есть (confirmationCode, arrivedAtPickupAt, completed/disputed —
из V3; couriers.location — из V2). Новые методы репозиториев:

```dart
// OrderRepository
Stream<TajGoOrder?> orderStream(String orderId);          // для трекинга
Stream<List<TajGoOrder>> recentOrdersStream(String customerId);
// = существующий запрос customerId+createdAt desc, limit 10,
//   фильтр client-side: completed/cancelled/disputed
// createOrder(...): + параметр comment
```

Композитный индекс не нужен — тот же запрос, что у activeOrderStream.
