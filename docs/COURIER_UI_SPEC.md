# TajGo — Courier MVP v0.3.0: дизайн-система и UI-спецификация

UI/UX-слой для курьерской части. Дополняет функциональную спеку
`docs/SPEC_CODEX_V3.md` (гео-гейт, код подтверждения, activeOrderId) — при
внедрении выполнять ВМЕСТЕ: V3 даёт логику, этот документ — внешний вид.
Архитектуру, Firebase-логику и main.dart НЕ менять. Существующие экраны
не удалять (актуальный список: SplashScreen, CustomerHomeScreen,
CourierHomeScreen, NewOrderMapScreen; RoleScreen и TajGoMapScreen удалены
в прошлых итерациях — НЕ восстанавливать).

## 1. Концепция дизайна

Бело-зелёный «Платформа TG+»: светлый фон, белые карточки, зелёная
градиентная шапка, лаймовое главное действие. Для курьера правило одно:
**на каждом экране ровно одна большая лаймовая кнопка — следующее действие**.
Всё остальное — вторично и не спорит с ней. Работа одной рукой: все действия
в нижней трети экрана, цели нажатия ≥ 48 px.

## 4. Экраны

### 4.1 CourierHomeScreen (обновить)

Сверху вниз:

1. **Шапка** (градиент darkGreen→green, скругление снизу 24):
   строка «TajGo Курьер · 📍 Худжанд» (белый 70%, 13) + `TajGoStatusPill`
   справа; крупно «Вы на линии 🟢» / «Готовы выйти на линию?» (белый w900);
   «📡 координаты передаются» (12, white70) когда трансляция активна;
   **кнопка-тумблер**: offline → белая кнопка «Выйти на линию» (текст
   darkGreen); online → контурная белая «Уйти с линии».
2. **Ряд статистики** — 4 × `TajGoStatCard` в ряд (или 2×2 на узких):
   💰 «N TJS» / Сегодня; 📦 «N» / Заказов; ⭐ «5.0» / Рейтинг;
   🏆 «100» / TajGo Score. Данные — из `TajGoCourier` (для «Заказов»
   добавить поле `ordersToday`, `increment(1)` в транзакции перехода
   в completed — та же транзакция, что и earningsToday).
3. **Активный заказ** (если есть): `TajGoOrderCard` на фоне `mint` +
   `TajGoOrderProgress` + строка статуса; тап → CourierOrderScreen.
   Лента при этом скрыта, вместо неё подпись «Завершите текущий заказ,
   чтобы принять следующий» (muted, по центру).
4. **Лента «Ожидающие заказы»** (только online и без активного заказа):
   `TajGoOrderCard` с кнопками в нижней части карточки (зона большого
   пальца): слева «Отказаться» (вторичная), справа «Принять»
   (`TajGoActionButton`, ~60% ширины). Offline → пустое состояние:
   иконка 🛵, «Выйдите на линию, чтобы видеть заказы».

Нижней tab-панели НЕТ (появится с ростом разделов) — навигация назад
стрелкой AppBar не нужна: шапка кастомная, назад — системный жест +

## 5. Состояния экранов

### CourierHomeScreen

| Состояние | Что видно |
|---|---|
| offline | серая пилюля, белая кнопка «Выйти на линию», пустое состояние ленты |
| online, нет заказов | зелёная пилюля + 📡, «Пока заказов нет» + иконка |
| online, есть waiting | лента карточек с Принять/Отказаться |
| есть активный заказ | mint-карточка с прогрессом, лента скрыта подсказкой |
| ошибка гео | SnackBar с текстом исключения, тумблер остаётся online |

### CourierOrderScreen (стадии = статусы Firestore + локальная «На месте»)

| Стадия | Статус | Заголовок | Главная кнопка |
|---|---|---|---|
| 1 | accepted | «Еду к точке забора» | «Я на месте» (пишет `arrivedAtPickupAt`, не блокирующее) |
| 2 | accepted + arrived | «Вы на месте» | «Забрал заказ» (гео-гейт 2 км из V3) |
| 3 | pickedUp | «Доставляю» | «Передал заказ» → диалог кода (V3) |
| 4 | delivered | «⏳ Ждём подтверждения клиента» | нет (текст-пояснение, лента заблокирована) |
| 5 | completed | «✅ Доставлено! +N TJS» | «К заказам» (pop) |
| — | disputed | «⚠️ Клиент сообщил, что не получил» | нет |

«Я на месте» — новое необязательное поле заказа `arrivedAtPickupAt`
(timestamp, пишется методом репозитория `markArrived`); статусную машину
V3 НЕ меняет и ничего не блокирует — это UX-чекпоинт (позже станет
уведомлением клиенту).

### Карточка заказа у клиента (для полноты статусов)

waiting «🔎 Ищем курьера» → accepted «🚴 Курьер принял» (+ после
`arrivedAtPickupAt` можно «Курьер на месте») → pickedUp «📦 Везём» + КОД →
delivered «Подтвердите получение» (кнопки) → completed (карточка исчезает) /
disputed «⚠️ Разбираемся».
