# TajGo Marketplace — инструкция администратора

Версия приложения: `1.5.0+30`. Раздел **Партнёры и товары** предназначен для
пилотной витрины Худжанда: Еда, Продукты и Цветы.

## Что видит клиент до наполнения Firestore

Если в выбранной категории нет активных партнёров или чтение временно
недоступно, приложение показывает локальную preview-витрину: 2 заведения и 6
товаров на каждую из трёх категорий. Названия помечены `Пример ·`.

Preview-карточки нельзя оформить в заказ. Администратор должен либо добавить
реальные данные, либо нажать **Опубликовать примеры**. Публикация создаёт/обновляет
6 тестовых партнёров и 18 товаров одной Firestore transaction и пишет
`admin_logs`. Повторный запуск идемпотентен по стабильным ID.

## Ручное наполнение

1. Войти аккаунтом с `users/{uid}.role = "admin"`.
2. Открыть **Админка → Партнёры и товары**.
3. Нажать добавление партнёра и заполнить название, категорию, адрес,
   координаты, минимум, доставку, рейтинг, время приготовления, часы работы и
   порядок сортировки.
4. Открыть карточку партнёра — появится его ассортимент.
5. Добавить товар: название, описание, HTTPS URL изображения, цена/старая цена,
   единица, доступность, видимость, популярность и порядок.
6. Для временного отключения используйте `isActive`, `isOpen`, `isAvailable`
   или `hidden`. Физического удаления из UI нет.

## Пакетный JSON

В разделе нажать **Импорт JSON**. Экран содержит редактируемый шаблон схемы 1.
Сначала нажать **Проверить**: запись в Firestore не выполняется. После dry-run
показываются партнёр, товары и предупреждения по координатам. Кнопка импорта
появляется только после успешной проверки и дополнительного подтверждения.

Корневая структура:

```json
{
  "schemaVersion": 1,
  "partner": {
    "id": "partner-id",
    "name": "Название",
    "category": "food",
    "address": "Худжанд",
    "latitude": 40.2833,
    "longitude": 69.6222,
    "minimumOrder": 30,
    "deliveryFee": 12
  },
  "products": [
    {
      "id": "product-id",
      "name": "Товар",
      "price": 25,
      "unit": "item"
    }
  ]
}
```

Допустимые категории: `food`, `groceries`, `flowers`. Единицы: `item`, `kg`,
`portion`, `bouquet`. ID содержат только латинские буквы, цифры, `_` и `-`.
Изображения — пустая строка или HTTPS URL. Один импорт содержит 1 партнёра и до
100 товаров. ID товара нельзя переиспользовать у другого партнёра.

## Безопасность и дальнейший Cloud-этап

Запись партнёров, товаров и admin logs разрешена только admin. Клиентские query
видят только `isActive == true` и `hidden == false`. Rules дополнительно
ограничивают поля, типы, цены, рейтинг и диапазоны.

Для закрытого пилота клиентский checkout повторно читает актуальные цены и
наличие в transaction. Для массового production следующий рекомендуемый этап —
перенести окончательный расчёт корзины и создание заказа в Firebase Callable
Function/App Check, добавить партнёрский кабинет, остатки, расписания и аудит
импорта. Это требует отдельного подтверждения владельца, Cloud billing и deploy;
в `1.5.0` ничего в Cloud автоматически не разворачивается.
