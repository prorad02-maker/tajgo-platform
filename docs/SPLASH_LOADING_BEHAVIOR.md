# Splash loading behavior

Splash имеет один динамический progress bar. Старые `splash_light.png` и
`splash_dark.png` используются как фон. Нижняя область со старой нарисованной
линией закрывается брендовой градиентной панелью, поэтому две линии одновременно
не видны.

## Последовательность

1. Flutter controller начинается с 0%.
2. За 1500 мс progress плавно приближается к 90%.
3. Параллельно выполняются anonymous auth и подготовка профиля.
4. Если bootstrap дольше, линия ждёт на 90% с мягкой пульсацией.
5. После bootstrap линия за 300 мс достигает 100%.
6. Только после завершения анимации открывается следующий экран.

Минимальная задержка splash — 1400 мс. Долгая инициализация ограничена
существующими timeout по 10 секунд для auth/profile. При ошибке progress
останавливается, становится красным и остаётся понятный текст.

## Проверка на телефоне

- видна ровно одна линия;
- заполнение начинается слева с нуля;
- линия реально движется;
- текст занимает фиксированную высоту и не прыгает;
- RoleScreen появляется только после 100%;
- проверить светлую и тёмную системную тему.
