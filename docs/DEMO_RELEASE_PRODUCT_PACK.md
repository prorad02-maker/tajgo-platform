Этот документ — продуктовый слой релиза: тексты, готовность демо, подача

## 1. Разбивка задачи v0.7.0: что куда

### Codex (код) — три задачи, в порядке приоритета

| # | Задача | Спека |
|---|---|---|
| 1 | Admin / Dispatch MVP: 5 экранов, AdminRepository (7 действий + admin_logs), вход по role==admin (+kDebugMode в debug), экран «Нет доступа» | `docs/ADMIN_UI_SPEC.md` — готова, обновлена ⚠️-предупреждением о задеплоенных rules |
| 2 | RouteService: `directRoute`/`roadRoute` → {distanceKm, etaMinutes, polylinePoints, isRoadRouteApproximation}; сейчас fallback-прямая; интерфейс под OSRM; подключить на 4 карточных экрана | ТЗ — §2 этого документа |
| 3 | Pricing v2: min 10, base 7, 3 TJS/км, округление до целого + тексты из §3 | ТЗ — §2 этого документа |

**Два критичных предупреждения для Codex** (из инцидента GPS v0.6.0):

1. Задеплоенные Firebase-правила строже локального `firestore.rules`.
   Любые НОВЫЕ поля в существующие переходы статусов — только с fallback
   (как в `updateLocation`) или после деплоя правил. Иначе сломается
   основной поток заказа.
2. Admin-действия под задеплоенными правилами получат permission-denied —
   это ожидаемо до деплоя. В приложении показывать честную ошибку
   («Нет прав — требуется обновление правил»), локальный `firestore.rules`
   дополнить `isAdmin()` по ADMIN_UI_SPEC §H. Деплой — только с явного
   разрешения владельца.

## 2. Краткие ТЗ для Codex (этапы 2–3)

### RouteService (`lib/core/services/route_service.dart`)

```dart
class TajGoRoute {
  final double distanceKm;      // Haversine, 1 знак
  final int etaMinutes;         // courierNavigationEtaMinutes(distanceKm)
  final List<LatLng> points;    // сейчас [from, to]
  final bool isRoadRouteApproximation; // сейчас всегда true
}
class RouteService {
  TajGoRoute directRoute({required LatLng from, required LatLng to});
  Future<TajGoRoute> roadRoute({...}) async => directRoute(...); // место для OSRM
}
```

Регистрация в TajGoScope. Экраны CourierOrderScreen, OrderTrackingScreen,
AdminOrderDetailsScreen, DispatchMapScreen берут линию A→B и км/ETA из
сервиса; при `isRoadRouteApproximation` показывают подпись из §3.
Живая дистанция «курьер → цель» остаётся прямым Haversine — это честно.

### Pricing v2 (`lib/core/services/pricing.dart`)

```dart
const num minimumPrice = 10; const num basePrice = 7; const num pricePerKm = 3;
num suggestedPrice(double km) => max(minimumPrice, (basePrice + pricePerKm*km).round());
```

Обновить юнит-тест: `suggestedPrice(0)==10; (1)==10; (2.5)==15; (10)==37`.
Цена остаётся редактируемой клиентом — формула только предзаполняет.

## 3. Продуктовые тексты (готовые строки, использовать дословно)

| Где | Текст |
|---|---|
| Клиент, детали заказа (под ценой) | «Цена рассчитана по расстоянию. В тестовом режиме может быть уточнена.» |
| Клиент/курьер, под линией маршрута | «Маршрут предварительный — показана прямая линия.» |
| Курьер, панель заказа | «Ваш доход за заказ: N TJS.» |
| Админ, entry-модуль на главной | «🛠 Управление TajGo» + бейдж «admin» (в debug — «debug») |
| Админ, экран без прав | «Нет доступа. Раздел доступен только администраторам TajGo.» |
| Админ, confirm «Отменить заказ» | «Заказ будет отменён для клиента и курьера. Укажите причину — она сохранится в журнале.» |
| Админ, confirm «Вернуть в waiting» | «Заказ снова увидят все курьеры. Текущий курьер будет снят с заказа. Продолжить?» |
| Админ, confirm «Завершить вручную» | «Заказ будет завершён, курьер получит оплату за заказ. Обычно это делается после разбора спора.» |
| Админ, confirm «Пометить спорным» | «Заказ будет передан на разбор. Курьер не сможет брать новые заказы до решения.» |
| Админ, ошибка прав | «Действие не выполнено: не хватает прав. Требуется обновление правил безопасности.» |

Тон: коротко, по-русски, без англицизмов в интерфейсе; последствия опасных
действий проговариваются в confirm-диалоге, не после.

## 5. Definition of Done для v0.7.0

1. Сквозной поток клиент→курьер→код→completed работает (регресс отсутствует).
2. Админка: все 5 экранов открываются, цифры дня совпадают с реальностью,
   4 действия над заказом и 2 над курьером работают (после деплоя правил)
   и пишут admin_logs.
3. RouteService подключён на 4 экранах, подпись «предварительный» видна.
4. Цены по новой формуле; тексты §3 стоят на местах.
5. `flutter analyze` / `flutter test` чистые; APK собирается.
6. DEMO_SCRIPT дополнен частью 5 и прогнан.

## 6. Известные ограничения демо (и как о них говорить)

| Ограничение | Формулировка для партнёров |
|---|---|
| Маршрут — прямая линия | «Маршрут предварительный; дорожная навигация подключается сервисом маршрутизации — интерфейс уже готов» |
| Phone Auth — тестовые номера (SHA не настроен) | «SMS-вход включается конфигурацией под боевой номер проекта» |
| Правила безопасности для админки требуют деплоя | не упоминать; задеплоить до показа |
| Трансляция GPS только при открытом приложении | «Фоновый режим — в дорожной карте; для демо не критично» |
| Поиск адреса — заглушка | «Точка на карте — основной сценарий; поиск в следующей версии» |

## 7. Что НЕ делать сейчас

- OSRM/GraphHopper, платные API карт, свой backend — интерфейс готов, интеграция позже.
- Push-уведомления, фоновая геолокация, чат, звонки.
- Web-дашборд, аналитика, бухгалтерия, автоназначение заказов, многоуровневые роли.
- Firebase deploy, SHA, боевой Phone Auth — отдельный финальный этап «под клиента».
- Новые pub-пакеты сверх уже разрешённых.
- Эксперименты с дизайном — RC: стабильность важнее новизны.
