# Локальные места Худжанда

Локальный справочник находится в `assets/data/khujand_places.json`. Он помогает
находить знакомые жителям короткие названия до обращения к интернет-геокодеру.
Текущие записи явно помечены как демо и не должны считаться проверенной базой.

## Как добавить место

Добавьте объект в JSON-массив:

```json
{
  "id": "place_unique_id",
  "title": "Название для результата поиска",
  "shortTitle": "Короткое название",
  "aliases": ["вариант 1", "вариант 2", "латинское написание"],
  "address": "Проверенный полный адрес",
  "lat": 40.000000,
  "lng": 69.000000,
  "category": "landmark"
}
```

Поддерживаемые категории: `cafe`, `shop`, `street`, `district`, `landmark`.

## Правила данных

1. Координаты проверяются владельцем на карте до добавления.
2. `id` уникален и после публикации не меняется.
3. В `aliases` добавляются разговорные, русские, таджикские и латинские варианты.
4. Кафе и магазины добавляются только с подтверждённой точкой входа.
5. Для улицы указывается центр улицы только как поисковая подсказка; пользователь
   всё равно уточняет дом и подъезд на карте.
6. Районы и ориентиры не должны автоматически становиться точкой заказа — после
   поиска пользователь всегда подтверждает положение пина.
7. Нельзя добавлять выдуманные координаты. Непроверенное место остаётся вне файла.

После изменения выполните `flutter pub get`, `flutter analyze` и проверьте поиск
на физическом телефоне.
