# База мест Худжанда: импорт и проверка

Источник приложения — `assets/data/khujand_places.json`. Текущие записи являются
демо-шаблонами и имеют `verified: false`. Перед пилотом координаты проверяются
человеком у реального входа, а не только визуально по карте.

## Полная схема записи

```json
{
  "id": "partner_cafe_001",
  "title": "Полное название",
  "shortTitle": "Короткое название",
  "aliases": ["разговорное имя", "таджикский вариант", "latin variant"],
  "address": "Проверенный адрес и ориентир",
  "district": "Район",
  "category": "cafe",
  "lat": 40.000000,
  "lng": 69.000000,
  "verified": true,
  "popularity": 80,
  "pinned": true,
  "isPartner": true,
  "partnerId": "partner_001",
  "tags": ["food", "pickup", "partner"],
  "notes": "Проверил: имя, дата, способ"
}
```

Категории: `market`, `cafe`, `shop`, `pharmacy`, `school`, `hospital`,
`landmark`, `street`, `district`, `mall`, `government`, `transport`.

## Правила качества

1. `id` уникален и стабилен. `partnerId` связывает место с будущим backend,
   но не содержит секретов.
2. `pinned` закрепляет важное место в поиске; не ставьте его массово.
3. `isPartner` используется только после согласия партнёра и проверки входа.
4. `verified: true` означает полевую проверку координаты. Дата/проверяющий — в
   `notes`.
5. Aliases включают русские, таджикские, разговорные и латинские варианты без
   ложных названий.
6. Popularity 0–100 задаёт порядок, а не точность.
7. Улицы и районы остаются подсказками: клиент всё равно подтверждает пин.

## Безопасный импорт

Проверить JSON редактором, затем выполнить `flutter analyze`, `flutter test` и
поиск на телефоне. Для партии 50+ мест сначала импортировать 5, пройти полевой
тест, затем добавлять остальное. Не публиковать непроверенные координаты как
реальные.
