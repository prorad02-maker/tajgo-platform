# Локальные места Худжанда

Локальный справочник находится в `assets/data/khujand_places.json`. Он помогает
находить знакомые жителям короткие названия до обращения к интернет-геокодеру.
Текущие записи явно помечены как демо и не должны считаться проверенной базой.

## Как добавить место

Добавьте объект в JSON-массив:

```json
{
  "id": "place_unique_id",
  "title": "Название для результата поиска",
  "shortTitle": "Короткое название",
  "aliases": ["вариант 1", "вариант 2", "латинское написание"],
  "address": "Проверенный полный адрес",
  "district": "Название района",
  "category": "landmark",
  "lat": 40.000000,
  "lng": 69.000000,
  "verified": false,
  "popularity": 50,
  "notes": "Кто и когда проверил точку"
}
```

Поддерживаемые категории: `market`, `cafe`, `shop`, `pharmacy`, `school`,
`hospital`, `landmark`, `street`, `district`, `mall`, `government`, `transport`.

## Правила данных

1. Координаты проверяются владельцем на карте до добавления.
2. `id` уникален и после публикации не меняется.
3. В `aliases` добавляются разговорные, русские, таджикские и латинские варианты.
4. Кафе и магазины добавляются только с подтверждённой точкой входа. После
   проверки поставьте `verified: true`; до проверки всегда `false`.
5. Для улицы указывается центр улицы только как поисковая подсказка; пользователь
   всё равно уточняет дом и подъезд на карте.
6. Районы и ориентиры не должны автоматически становиться точкой заказа — после
   поиска пользователь всегда подтверждает положение пина.
7. `popularity` — целое число 0–100. Оно задаёт порядок популярных мест, но не
   влияет на точность координат.
8. В `notes` запишите источник и дату проверки. Поле не показывается клиенту.
9. Нельзя добавлять выдуманные координаты как точные. Демо-точки имеют
   `verified: false` и явную пометку в названии/заметке.

## Как не сломать JSON

- между объектами нужна запятая, после последнего объекта — нет;
- строки заключаются в двойные кавычки;
- `lat`, `lng`, `popularity` пишутся без кавычек;
- `verified` принимает только `true` или `false`;
- перед коммитом откройте файл в редакторе с JSON-проверкой и запустите сборку.

После изменения выполните `flutter pub get`, `flutter analyze` и проверьте поиск
на физическом телефоне.
